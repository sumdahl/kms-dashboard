# Fix Admin Sidebar Visibility

## Problem
The admin navigation section (Roles / Assign Role) is not appearing for admin users. The frontend JavaScript correctly checks for `payload.is_admin` at `sidebar.html:416` but the JWT token being issued does **not** include the `is_admin` claim in its payload.

Only the user `admin@example.com` has `is_admin = true` in the database. All other users correctly have this flag set to false.

## Root Cause
✅ Backend correctly includes `is_admin` boolean in the JWT claims. ❌ **Actual issue**: The Rust `serde` serializer emits snake_case fields by default. The JWT token contains `is_admin` but the JavaScript frontend at `sidebar.html:416` is checking for `payload.is_admin` which **does exist** but had an invalid cache / old token.

**Critical secondary issue**: The admin routes `/roles` and `/assign` **do not have RBAC protection** - they are currently publicly accessible to all authenticated users.

## Implementation Plan

✅ **Confirmed backend is already correct**:
  - `src/auth/jwt.rs:7-26` properly receives `is_admin` parameter and includes it in Claims
  - `src/handlers/auth.rs:58` correctly passes `user.is_admin` from database record
  - `src/models/auth.rs:9` has `is_admin: bool` field with default serde serialization

1. **Fix 1: JavaScript Frontend (Primary)**
   - Clear localStorage cached token on logout
   - **Workaround**: Fix JS to properly decode token field. NOTE: `serde` serializes `is_admin` as `"is_admin"` (snake_case) in JWT tokens which matches perfectly with the JS code.

2. **Fix 2: Secure Admin Routes (Critical)**
   - **Add admin middleware protection** to `/roles`, `/assign`, `/users` routes
   - All admin routes are currently unprotected - any authenticated user can access them directly via URL even if sidebar is hidden

3. **Test validation**
   - Logout and login again with admin@example.com: admin section appears
   - Login as normal user: admin section remains hidden
   - Attempt direct URL access to `/roles` as non-admin: returns 403 Forbidden
   - Verify JWT token payload contains correct boolean value

## Files to modify
- `templates/partials/sidebar.html` - Fix token handling and add logout clearing
- `src/routes/admin.rs` - Add admin auth middleware to all admin routes
