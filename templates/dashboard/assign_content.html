Assigning a role grants the user all permissions associated with it. If the user already has this role, the assignment will be updated with the new expiry.

Assignment Notes

  • Temporary Access: Set a duration for time-sensitive tasks or third-party audits.

  • Inheritance: Users will immediately receive all resource permissions defined in the role.

  • Updates: You can re-assign the same role to update or clear the expiration time.

Role not listed?

If you don't see the role you need, you may need to create it first in the Roles management section.
